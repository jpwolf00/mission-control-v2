## UX Design Specification

### 1. Revision Actions Card

**Location:** Story detail page, below attachments section  
**Visibility:** Stories with status `completed` or `active`

#### UI Components

```tsx
<Card sx={{ mt: 3, border: '1px solid', borderColor: 'primary.main' }}>
  <CardHeader 
    title="Review Actions" 
    subheader="Accept this story as complete or send it back for revisions"
  />
  <CardContent>
    {/* Action Buttons */}
    <Button variant="contained" color="success" startIcon={<CheckCircleIcon />}>
      Accept Final
    </Button>
    <Button variant="outlined" color="warning" startIcon={<ReplayIcon />}>
      Request Revisions
    </Button>
  </CardContent>
</Card>
```

#### Interactions

1. **Accept Final**
   - Immediate POST to `/api/v1/stories/[id]/revisions` with `action: 'accept_final'`
   - Shows success alert
   - Refreshes story state
   - Changes story status to `accepted`

2. **Request Revisions**
   - Opens modal dialog
   - User selects target gate (dropdown)
   - User enters revision notes (optional)
   - Submits POST to `/api/v1/stories/[id]/revisions` with `action: 'request_revision'`
   - Story status changes to `in_progress`
   - Story dispatched to selected gate

#### Gate Selection Dropdown

Options (in order):
1. Design (Architect)
2. UX Review (UI Designer)
3. Build (Implementer) ← **Default**
4. Review (Reviewer A)
5. Deploy (Operator)
6. Validate (Reviewer B)

## User Flows

### Flow 1: Accept Final Output

```
User views completed story
    ↓
Clicks "Accept Final" button
    ↓
POST /api/v1/stories/[id]/revisions { action: 'accept_final' }
    ↓
Story status → 'accepted'
    ↓
Success alert shown
    ↓
Story archived (read-only)
```

### Flow 2: Request Revisions

```
User views completed story
    ↓
Clicks "Request Revisions" button
    ↓
Modal dialog opens
    ↓
User selects target gate (default: implementer)
    ↓
User enters revision notes (optional)
    ↓
Clicks "Request Revisions" in dialog
    ↓
POST /api/v1/stories/[id]/revisions { action: 'request_revision', targetGate, description }
    ↓
Story status → 'in_progress'
    ↓
Downstream gates cleared
    ↓
Story dispatched to target gate
    ↓
Success alert + page refresh
```

### Flow 3: Add Comment

```
User views story (any status)
    ↓
Types comment in text field
    ↓
Clicks "Post Comment"
    ↓
POST /api/v1/stories/[id]/comments { content }
    ↓
Comment appears in list
    ↓
Input cleared
```
