## Usage Guidelines

### File Naming Convention
```
[TYPE]-[NUMBER]-[DESCRIPTION]-[SIZE].jpg
Examples:
HERO-001-LivingRoom-1920x1080.jpg
SVC-003-SmartHome-800x600.jpg
PROJ-001-Tribeca-1200x800.jpg
```

### Recommended Sizes
- **Hero banners**: 1920x1080 (16:9) or 2560x1440 for retina
- **Service cards**: 800x600 (4:3) or 600x400
- **Project cards**: 600x400 (3:2)
- **CTA backgrounds**: 1920x1080 with overlay

### Text Overlay Tips
- Use dark gradient overlay (20-40% opacity) on bright images
- White text on dark areas, dark text on light areas
- Leave negative space in composition for text
- Test readability at actual display size

### Consistency Notes
- Maintain consistent color temperature across all images (warm, 4500-5500K)
- Use similar contrast levels
- Avoid overly saturated or HDR looks
- Keep technology subtle—lifestyle first, equipment second

## Prompt Engineering Tips

1. **Start with the prompt above** as a base
2. **Adjust for your tool**: 
   - Midjourney: Keep `--ar`, `--style raw`, `--q 2` parameters
   - DALL-E 3: Remove parameters, describe aspect ratio in words
   - NanoBanana: Follow its specific syntax
3. **Iterate on faces**: If generating people, you may need multiple attempts for natural expressions
4. **Control consistency**: Use seed values (if supported) to maintain style across variations
5. **Upscale**: Generate at highest resolution available, upscale if needed for large displays
